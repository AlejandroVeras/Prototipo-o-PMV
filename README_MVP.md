# InventoryAI - MVP (Producto Mínimo Viable)

## 📋 Resumen del Proyecto

**InventoryAI** es un SaaS de automatización de inventarios con visión artificial. Este MVP valida la hipótesis de que las PYMES comerciales e industriales pagarán por una solución que automatice sus inventarios con precisión del 98%+ usando cámaras estándar.

## 💡 Hipótesis Validadas en el MVP

### ✅ Hipótesis 1: El Problema es Real
**Validado:** 80% de los 10 clientes entrevistados confirmó que es crítico.
- Dedican 3-8 horas/día a conteos manuales
- Errores generan pérdidas de RD$5-12k/mes
- Falta de visibilidad para decisiones de compra

### ✅ Hipótesis 2: Disposición a Pagar
**Validado:** 80% estaría dispuesto a pagar $15-30/mes.
- Pagarían por precisión (98%+)
- Pagarían por ahorro de tiempo (5+ horas/semana)
- Pagarían por control de pérdidas

### ❓ Hipótesis a Validar en MVP:
- ¿Aceptarán usar cámaras estándar (vs equipos costosos)?
- ¿Será suficiente 98% de precisión para tomar decisiones?
- ¿Pagarán $20-25 (precio sweet spot) vs $15 o $50?
- ¿Integración ERP es feature V1 o V2?

## 📝 Preguntas Frecuentes

**P: ¿Los datos del dashboard son reales?**
A: Los datos son **simulados pero realistas**. Basados en clientes validados, pero no son datos de un cliente específico.

**P: ¿Esto es el producto final?**
A: No. Este es un MVP (Producto Mínimo Viable) para validar concepto. El producto real tendrá:
- Algoritmo real de visión artificial (procesamiento de imágenes)
- Captura real desde cámaras CCTV
- Integración técnica con ERP
- Base de datos persistente

**P: ¿Cuánto tiempo tomó construir esto?**
A: El MVP (landing + dashboard) se construyó en menos de 8 horas usando HTML5/CSS/JS. El objetivo era validar concepto rápidamente, no construir el producto final.

**P: ¿Puedo modificar el MVP?**
A: Sí. El código es HTML/CSS/JavaScript simple. Puedes personalizar colores, textos, planes de precios, etc.
